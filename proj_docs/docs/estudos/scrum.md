# Relatório Técnico  
## Aplicação do Método Scrum em Projeto de Monitoramento de Coerência entre Discurso e Votação

---

## 1. Introdução

Este relatório apresenta um estudo técnico sobre a aplicação do método ágil Scrum em um projeto de desenvolvimento de software voltado ao monitoramento de coerência entre discursos políticos e votações legislativas. O sistema tem como objetivo analisar, por meio de técnicas de processamento de linguagem natural, a relação entre o posicionamento declarado por parlamentares e suas decisões em plenário.

O projeto possui alta complexidade técnica, envolvendo coleta de dados públicos, processamento de grandes volumes de texto, análise semântica e visualização de informações. Dessa forma, a utilização do Scrum é adequada por permitir entregas incrementais, adaptação contínua e melhor organização de equipes multidisciplinares.

---

## 2. Visão Geral do Scrum

O Scrum é um framework ágil baseado em ciclos iterativos e incrementais chamados de Sprints. Cada Sprint resulta em uma entrega funcional do sistema, permitindo validação contínua e ajustes ao longo do desenvolvimento.

Os principais elementos do Scrum são:
- Papéis definidos
- Backlog do produto
- Sprints com duração fixa
- Reuniões periódicas (eventos)
- Entregas incrementais

---

## 3. Estrutura da Equipe (6 Pessoas)

Para este projeto, a equipe é organizada da seguinte forma:

- **Product Owner (1 pessoa):**
  Responsável por definir os requisitos, priorizar o backlog e garantir que o produto atenda aos objetivos do projeto.

- **Scrum Master (1 pessoa):**
  Responsável por garantir a correta aplicação do Scrum, remover impedimentos e facilitar a comunicação entre os membros.

- **Equipe de Desenvolvimento (4 pessoas):**
  - Desenvolvedor Backend (API e integração)
  - Especialista em NLP/IA (análise semântica)
  - Desenvolvedor Frontend (dashboard)
  - Engenheiro de Dados (coleta e tratamento de dados)

Essa divisão permite cobrir todas as áreas críticas do projeto de forma equilibrada.

---

## 4. Definição do Produto

### 4.1 Objetivo

Desenvolver um sistema capaz de analisar a coerência entre discursos políticos e votações legislativas, gerando indicadores e visualizações acessíveis.

### 4.2 Funcionalidades Principais

- Coleta de dados de votações e discursos
- Processamento e análise de texto
- Cálculo de índice de coerência
- Visualização em dashboard interativo
- Histórico de posicionamento por parlamentar

---

## 5. Product Backlog

O Product Backlog consiste na lista priorizada de funcionalidades e tarefas do projeto.

### 5.1 Épicos

- Coleta de Dados
- Processamento de Linguagem Natural
- Análise de Coerência
- Desenvolvimento de API
- Desenvolvimento de Dashboard

### 5.2 Exemplos de User Stories

- Como usuário, quero visualizar o histórico de votos de um parlamentar.
- Como pesquisador, quero comparar discurso e voto para identificar inconsistências.
- Como usuário, quero filtrar dados por período e partido.

---

## 6. Eventos do Scrum

### 6.1 Sprint Planning
Reunião para definição das tarefas da Sprint, com base na priorização do backlog.

### 6.2 Daily Scrum
Reuniões diárias curtas para acompanhamento do progresso, identificação de impedimentos e alinhamento da equipe.

### 6.3 Sprint Review
Apresentação das entregas realizadas ao final da Sprint, com coleta de feedback.

### 6.4 Sprint Retrospective
Avaliação do processo de trabalho, identificando pontos de melhoria para as próximas Sprints.

---

## 7. Ferramentas de Apoio

Para a execução do Scrum, recomenda-se o uso das seguintes ferramentas:

- Gerenciamento de tarefas: Trello, Jira ou Notion
- Controle de versão: Git e GitHub
- Comunicação: Discord ou Slack
- Documentação: Markdown ou Google Docs

---

## 8. Benefícios da Aplicação do Scrum

A aplicação do Scrum neste projeto proporciona:

- Entregas contínuas e incrementais
- Melhor adaptação a mudanças
- Maior transparência no desenvolvimento
- Distribuição eficiente de tarefas
- Redução de riscos técnicos

Além disso, permite validar rapidamente partes críticas do sistema, como o modelo de análise semântica.

---

## 9. Desafios e Limitações

Alguns desafios esperados incluem:

- Integração de dados de diferentes fontes
- Complexidade na análise de linguagem política
- Definição precisa de métricas de coerência
- Dependência de dados públicos inconsistentes

Esses fatores exigem constante adaptação e validação ao longo do projeto.

---

## 10. Conclusão

A aplicação do método Scrum em uma equipe de seis pessoas é adequada para o desenvolvimento do sistema proposto, pois permite organização eficiente, divisão clara de responsabilidades e evolução progressiva do produto. Dada a complexidade do projeto, a abordagem iterativa facilita a validação contínua das soluções desenvolvidas, especialmente no que se refere à análise de dados e à implementação de modelos de linguagem.

O uso do Scrum contribui não apenas para a entrega do sistema, mas também para a formação técnica da equipe, promovendo boas práticas de engenharia de software e trabalho colaborativo.
